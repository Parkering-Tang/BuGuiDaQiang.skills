# schema-guard

> 数据安全审查：保护数据库和数据结构不被危险修改

## 触发条件

- 任何涉及数据库/存储的变更
- 任何涉及类型定义的变更
- 任何涉及 API 契约的变更
- 任何涉及配置文件中数据结构的变更

---

## 核心职责

1. **识别 schema 变更** - 检测是否涉及数据结构修改
2. **评估风险** - 判断变更对现有数据的影响
3. **阻止危险操作** - 拒绝不安全的类型转换
4. **提供替代方案** - 推荐更安全的实现方式

---

## 危险操作检测

### 🚨 高危操作（必须阻止或警告）

| 操作 | 风险 | 建议 |
|------|------|------|
| 类型收窄 | 数据丢失 | 先清理数据再改类型 |
| 类型扩展（无兼容） | 现有代码出错 | 保持兼容或迁移代码 |
| 删除字段 | 数据丢失 | 先标记废弃 |
| 重命名字段 | 现有代码出错 | 添加别名或迁移 |
| 存储不兼容类型 | 数据损坏 | 使用新字段 |

### ⚠️ 中危操作（需要确认）

| 操作 | 风险 | 建议 |
|------|------|------|
| 添加字段 | 需要处理默认值 | 确认默认值策略 |
| 修改字段约束 | 可能影响现有数据 | 检查现有数据合规性 |
| 索引变更 | 性能影响 | 评估查询影响 |

## 类型兼容性矩阵

### 安全的变更

| 从 | 到 | 风险 |
|---|---|------|
| string | string (更宽松) | 低 |
| number | number (更宽松) | 低 |
| 任意 | 可空 | 低 |
| 枚举 | string | 中（需确认值范围）|

### 危险的变更

| 从 | 到 | 风险 |
|---|---|------|
| string | number | 高（需验证所有值可转换）|
| boolean | string | 高（现有逻辑会失效）|
| 枚举 | 不同枚举 | 高（值映射问题）|
| 任意 | 必填 | 高（现有数据可能为空）|

### 破坏性变更

| 操作 | 风险等级 | 处理 |
|------|---------|------|
| 删除字段 | 🚨 破坏性 | 禁止，建议废弃流程 |
| 重命名字段 | 🚨 破坏性 | 禁止，建议别名或新字段 |
| 改变主键 | 🚨 破坏性 | 禁止，需要完整迁移计划 |

## 替代方案库

### 场景：用户想存储更丰富的状态值

❌ 危险：直接改布尔字段为字符串
```
status: boolean → status: string // 危险！
```

✅ 安全：添加新字段或使用枚举
```
// 方案 A：添加新字段
status: boolean      // 保持不变
statusText: string   // 新增显示文本

// 方案 B：使用枚举
status: 'active' | 'inactive' | 'pending'
```

### 场景：用户想存储中文

❌ 危险：直接在原字段存中文
```
type: 'user' → type: '普通用户' // 危险！破坏现有逻辑
```

✅ 安全：显示层映射
```
// 数据库保持不变
type: 'user'

// 前端显示映射
const typeLabels = {
  'user': '普通用户',
  'admin': '管理员'
}
```

### 场景：用户想删除字段

❌ 危险：直接删除
```
删除 oldField  // 数据丢失
```

✅ 安全：标记废弃 → 迁移 → 删除
```
// Step 1: 标记废弃
oldField: string @deprecated

// Step 2: 确认无使用后迁移数据

// Step 3: 安全删除
```

### 场景：用户想改变字段类型

❌ 危险：直接改类型
```
age: string → age: number  // 现有 "25" 数据无法使用
```

✅ 安全：新字段 + 迁移脚本
```
// Step 1: 添加新字段
age: string
ageNum: number | null

// Step 2: 创建迁移脚本，将 age 的值转换到 ageNum

// Step 3: 更新代码使用新字段

// Step 4: 废弃旧字段
```
