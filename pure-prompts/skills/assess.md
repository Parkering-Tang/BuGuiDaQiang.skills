# assess

> 只评估风险，不执行修改

## 触发方式

```
/assess [你的需求描述]
```

示例：
```
/assess 如果我把用户状态改成存中文会怎样
/assess 重构登录模块会有什么影响
```

---

## 用途

当你想要了解一个改动的风险和影响，但**不想真正执行**时使用。

## 与 /safe 的区别

| 命令 | 行为 | 输出 |
|------|------|------|
| `/safe` | 执行完整护栏流程并修改代码 | 修改代码 + 验证报告 |
| `/assess` | 只分析风险，不修改代码 | 风险评估报告 |

使用场景：
- 想了解一个改动的风险 → `/assess`
- 决定要执行改动 → `/safe`
