# scope-guard

> 范围守护：确保实现不会超出确认的范围

## 触发条件

在 clarify 确认后，对于 moderate 及以上复杂度的请求触发。

---

## 核心职责

1. **定义边界** - 明确哪些可以做，哪些不可以做
2. **识别越界** - 检测实现意图是否超出原定范围
3. **阻止蔓延** - 拒绝"顺便"做未确认的改动

---

## 边界定义规则

### ✅ 允许的操作

来自 clarify 阶段确认的内容：
- 明确确认的文件修改
- 实现确认功能所需的必要改动
- 自动生成的辅助代码（如类型定义）

### ❌ 不允许的操作

**严格禁止**：
- "顺便"重构代码
- 修改未在 affected_files 中的文件
- 添加未确认的新功能
- 修改配置文件（除非明确确认）
- 修改数据库 schema（除非通过 schema-guard）
- 修改 API 契约（除非明确确认）

### ⚠️ 需要额外确认

- 发现需要修改更多文件才能完成功能
- 发现原方案不可行，需要调整
- 发现依赖项需要更新

---

## "顺便"行为禁止清单

以下行为**严格禁止**，即使你认为"这样做更好"：

| 禁止行为 | 原因 |
|---------|------|
| ❌ 顺便重构周围代码 | 超出范围 |
| ❌ 顺便统一命名风格 | 超出范围 |
| ❌ 顺便优化性能 | 超出范围 |
| ❌ 顺便修复其他 bug | 超出范围 |
| ❌ 顺便添加类型注解 | 超出范围 |
| ❌ 顺便更新依赖版本 | 超出范围 |
| ❌ 顺便改进错误处理 | 超出范围 |
| ❌ 顺便添加日志 | 超出范围 |
| ❌ 顺便优化 import 顺序 | 超出范围 |
| ❌ 顺便添加 TODO 注释 | 超出范围 |

**判断标准**：

```
问自己：如果我不做这个修改，功能能正常工作吗？

✅ 能 → 这是"顺便"修改 → 禁止
❌ 不能 → 这是必要修改 → 允许
```

---

## 越界检测

在实现过程中，如果检测到以下情况，**必须暂停并询问**：

```
⚠️ 范围变更提示

在实现过程中发现：
• 需要修改 [文件名]，但这不在原定范围内
• 原因：[为什么需要修改]

原定范围：[文件列表]

是否允许扩展范围？（y/n）
或者是否需要重新评估方案？
```

---

## 执行检查清单

在执行任何修改前，检查：

```
修改前检查：
□ 这个文件在允许列表中吗？
□ 这个修改是功能必需的吗？
□ 这个修改不是"顺便"做的吗？
□ 我已经读取并分析了文件风格吗？
□ 新代码会遵循现有风格吗？

如果任何一项为否，暂停并处理。
```

## 示例

### 正常情况

**输入**：
```
请求：添加记住密码复选框
涉及文件：src/Login.tsx, src/services/auth.ts
```

**输出**：
```
┌─────────────────────────────────────────────────────┐
│ 🎯 实现范围边界                                      │
├─────────────────────────────────────────────────────┤
│ ✅ 允许的操作：                                      │
│ • 修改 src/Login.tsx - 添加复选框 UI                │
│ • 修改 src/services/auth.ts - 添加 rememberMe 参数 │
│ • 新建辅助函数文件（如需要）                         │
│                                                     │
│ ❌ 不允许的操作：                                    │
│ • 重构其他组件                                      │
│ • 修改 API 接口签名                                 │
│ • 修改数据库结构                                    │
│                                                     │
│ ⚠️ 需要额外确认：                                    │
│ • 如果需要修改全局状态管理                          │
│ • 如果需要修改其他页面                              │
└─────────────────────────────────────────────────────┘
```

### 检测到越界

**场景**：实现过程中发现需要修改 store

**输出**：
```
⚠️ 范围变更提示

在实现过程中发现：
• 需要修改 src/store/user.ts
• 原因：登录状态持久化需要更新 store

原定范围：src/Login.tsx, src/services/auth.ts

影响评估：
• 可能影响其他使用 store 的组件
• 需要额外测试

是否允许扩展范围？
1. ✅ 允许，继续实现
2. ❌ 不允许，寻找替代方案
3. ⏸️ 暂停，重新评估整体方案
```
