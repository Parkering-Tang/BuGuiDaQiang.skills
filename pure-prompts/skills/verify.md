# verify

> 验证完成：没有验证证据就不能说"完成了"

## 触发条件

在 implement 完成后触发。

---

## 核心职责

1. **运行构建** - 确保代码可以编译
2. **运行测试** - 确保功能没有被破坏
3. **解析结果** - 给用户清晰的验证报告
4. **提供回滚** - 如果验证失败，提供恢复选项

---

## 项目类型检测

执行以下命令检测项目类型：

```bash
# 检测项目类型
ls package.json 2>/dev/null && echo "node"
ls pyproject.toml 2>/dev/null && echo "python"
ls Cargo.toml 2>/dev/null && echo "rust"
ls go.mod 2>/dev/null && echo "go"
```

| 项目类型 | 检测文件 | 构建命令 | 测试命令 |
|---------|---------|---------|---------|
| Node.js | package.json | `npm run build` | `npm test` |
| Python | pyproject.toml | - | `pytest` |
| Rust | Cargo.toml | `cargo build` | `cargo test` |
| Go | go.mod | `go build` | `go test ./...` |

---

## 验证命令

### Node.js 项目

```bash
# 检查 package.json 中的脚本
cat package.json | grep -A 20 '"scripts"'

# 运行构建（如果存在）
npm run build 2>&1

# 运行测试（如果存在）
npm test 2>&1

# 运行类型检查（如果是 TypeScript）
npx tsc --noEmit 2>&1

# 运行 lint（如果存在）
npm run lint 2>&1
```

### Python 项目

```bash
# 运行测试
pytest 2>&1

# 运行类型检查（可选）
mypy . 2>&1

# 运行 lint（可选）
ruff check . 2>&1
```

### Rust 项目

```bash
# 构建
cargo build 2>&1

# 运行测试
cargo test 2>&1

# 运行 lint
cargo clippy 2>&1
```

### Go 项目

```bash
# 构建
go build ./... 2>&1

# 运行测试
go test ./... 2>&1
```

## 验证失败处理

### 构建失败

```
❌ 构建失败

可能的原因：
1. 语法错误
2. 类型错误
3. 依赖缺失
4. 配置问题

建议操作：
1. 查看错误信息的具体位置
2. 检查是否引入了未安装的依赖
3. 检查是否有拼写错误

回滚选项：
cp -r .safe-vibecoding/snapshots/snap-20260415-1030/* .
```

### 测试失败

```
❌ 测试失败

可能的原因：
1. 新代码与现有测试不兼容
2. 测试期望值需要更新
3. 功能实现有问题

建议操作：
1. 分析失败测试的具体断言
2. 判断是代码问题还是测试需要更新
3. 如果是代码问题，修复后重新验证
4. 如果是测试需要更新，确认后更新测试

回滚选项：
cp -r .safe-vibecoding/snapshots/snap-20260415-1030/* .
```

## 完成标准

```
□ 构建成功（或跳过，如果无构建命令）
□ 测试通过（或跳过，如果无测试命令）
□ 变更记录已生成
□ 快照 ID 已记录
□ 用户已知晓结果
```

## 项目无测试时的处理

如果项目没有测试：

```
⚠️ 未检测到测试配置

项目类型：Node.js
未找到：npm test 脚本

验证结果：
├── 构建：✅ 通过
└── 测试：⏭️ 跳过（项目未配置测试）

建议：
1. 手动测试变更的功能
2. 检查相关页面是否正常
3. 考虑添加测试

变更已应用，但建议手动验证。
```
